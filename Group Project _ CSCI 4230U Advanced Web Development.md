# CSCI 4230U Advanced Web Development

# Project 

For this project, you will work in groups of two to develop a web application that encompasses various aspects of modern web development. Each group will choose a unique theme or problem to solve, allowing you to showcase your creativity and technical skills. This project is an excellent opportunity to build something meaningful and add a substantial piece of work to your portfolio, which can greatly enhance your career prospects in the field of web development.

You are free to use any technology, language, and framework (the stack) for this project.

## Register

In Canvas, register your group (people \> groups).  
Submit by the deadline.

## Deliverables

* Access to GitHub/GitLab repository (public repository).   
* Clear instructions on how to build and run the project with Docker.  
* Live version of the project.

## Project requirements/assessment

### API Design and Integration (15%)

* Clear and well-structured API endpoints.  
* Proper use of HTTP methods (GET, POST, PUT, DELETE).  
* Integration with at least one external API.  
* Documentation of the API endpoints.

### Architectural Design (15%)

* Implementation of chosen architecture (e.g., Microservices, Monolithic, MVC).  
* Proper separation of concerns (frontend, backend, database).  
* Diagram and explanation of the chosen architecture (UML package and class diagrams).

### Authentication and Security (10%)

* Implementation of secure user authentication (e.g., OAuth, JWT, etc).  
* Protection of sensitive routes with proper authorization checks.  
* Handling of user sessions and secure storage of credentials.

### Database Design and ORM (10%)

* Database schema design.  
* Use of an ORM for database interactions (e.g., SQLAlchemy, Django ORM).  
* Implementation of CRUD operations.  
* Use of relationships between data entities.

### Testing and Quality Assurance (10%)

* Unit tests covering core functionalities.  
* Integration tests for API endpoints.  
* End-to-end tests with Selenium.

### Deployment and DevOps (10%)

* Deployment to a cloud platform or a general-purpose server.  
* Use of Docker for deployment consistency.  
* Setup of a CI/CD pipeline.  
* Documentation of deployment process.  
* Use of HTTPS with either HTTP v2 or HTTP v3. 

### Performance Optimization (5%)

* Analysis of application performance using https://pagespeed.web.dev/.

### Version Control and Collaboration (5%)

* Use of Git and GitHub/GitLab for version control.  
* Regular commits with clear messages.  
* Properly documented README file with project setup instructions.

### Code Quality and Documentation (5%)

* Proper use of comments and docstrings.  
* Adherence to coding standards and best practices (e.g., PEP 8 for Python).  
* Comprehensive project documentation, including setup, usage, and architecture.

### Presentation and Demonstration (15%)

* Clear and organized presentation of the project.  
* Demonstration of key features.  
* Explanation of design decisions and challenges faced.  
* Q\&A session to assess understanding and depth of knowledge.

## Project ideas

These are just a few ideas. You can propose new project ideas.

* E-Commerce Platform: Integrate with payment gateways like Stripe.   
* Social Networking Platform: Use real-time features (e.g., chat) with WebSockets.  
* Interactive Quiz Game: A web-based quiz game where users can select categories and difficulty levels. The game could integrate external APIs for questions, like Open Trivia DB.  
* Fitness Tracker: A health and fitness tracking app that integrates with external APIs for step counting or workout logging. It could include features like goal setting and progress tracking.  
* Recipe Finder: An app that allows users to search for recipes based on ingredients they have at home, using an external recipe API.  
* Virtual Pet Game: A game where users care for a virtual pet, feeding it, playing with it, and monitoring its health. The app could integrate with external APIs for added features like weather changes affecting the pet's environment.  
* Event Management System: A platform for organizing events, with features like ticket sales, attendee management, and calendar integration.  
* Language Learning App: A tool that helps users learn a new language by integrating with translation APIs and offering quizzes, flashcards, and daily challenges.  
* Movie Recommendation System: An app that recommends movies based on user preferences and integrates with external APIs like IMDb for movie details and reviews.  
* Budget Tracker: A personal finance app that helps users track expenses, create budgets, and visualize spending patterns, with optional integration of banking APIs for automated updates.  
* Travel Planner: A travel planning tool that helps users organize trips, find accommodations, and explore attractions using external travel APIs.  
* Online Marketplace: A platform where users can buy and sell products, with features like user reviews, payment gateway integration, and inventory management.  
* Real-Time Chat Application: A chat app using WebSockets for real-time communication, with user authentication and chat history stored in a database.  
* Weather-based Game: A game that changes its difficulty or environment based on the real-time weather of the player's location, using weather APIs.
